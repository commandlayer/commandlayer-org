Canonical agent descriptors for CommandLayer verbs. Each JSON document binds an ENS name, supported verbs, schema versions, and x402 entrypoints.

Commons & early Commercial agents
One JSON per ENS agent
Path pattern: /agent-cards/v1.0.0/<tier>/<ens>.json

What an Agent Card describes

Agent Cards don’t execute anything. They describe who an agent is, what verbs it offers, and how to reach it in a way any A2A client can understand.

Identity

ENS name, owner, and contact information that defines who operates the agent.

Capabilities

Supported verbs plus pinned Commons or Commercial schema versions the agent speaks.

Execution

x402 entrypoints for sending *.request payloads into the agent for actual execution.
