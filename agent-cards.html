Agent Cards (v1.0.0)

Canonical JSON descriptors for CommandLayer agents. Each card binds an ENS name, supported verbs, schema versions, and x402 entrypoints into one machine-readable document.

Commons & early Commercial verbs
One JSON per agent
Path pattern: /agent-cards/v1.0.0/<tier>/<ens>.json

What an Agent Card contains

Agent Cards are designed for discovery and interoperability. They don’t execute anything — they just describe it, in a way any A2A client can read.

Identity & ownership

ENS name, owner, contact, and metadata about who operates the agent.

Verbs & schemas

Which verbs are supported and which @commandlayer/protocol-commons or Commercial schema versions they speak.

x402 entrypoints

Where to send *.request payloads for execution, including network IDs and routing info.
