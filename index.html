What CommandLayer does

CommandLayer doesn’t replace agents, wallets, or LLMs. It gives them a shared command language so any client can understand what “summarize”, “checkout”, or “refund” actually mean.

Defines the verbs

CommandLayer specifies verbs and their JSON schemas so “analyze.request” or “checkout.receipt” are unambiguous across agents and platforms.

Aligns with x402 & ERC-8004

Verbs map cleanly into x402 envelopes and are discoverable by ERC-8004 agents, keeping payment, routing, and discovery interoperable.

Binds ENS to execution

Agent Cards bind ENS names, verb support, schema versions, and x402 endpoints, so clients know exactly who can serve which verbs.

How builders use CommandLayer

Three touch points. No lock-in. No magic.

1. Import the schemas

Use @commandlayer/protocol-commons for the 10 universal verbs. Validate A2A payloads under strict JSON Schema 2020-12.

2. Publish an Agent Card

Describe your agent: ENS name, supported verbs, schema versions, and x402 entrypoints. Let clients discover you trustlessly.

3. Wire x402 + ERC-8004

Expose your verbs via x402 endpoints and register under ERC-8004 so other agents can call you without custom glue code.
