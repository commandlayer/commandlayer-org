Where the semantic layer is going

From 10 Commons verbs to hundreds that define the agent economy.

CommandLayer starts as a small, stable verb set, then expands into Commercial economic flows, Agent Cards, and registries wired into x402 and ERC-8004. This roadmap shows what’s already shipped, what’s being wired now, and what’s staged next.

Commons v1.0.0 · Live Commercial verbs · In development Hundreds of verbs staged

How to read this roadmap

The focus is simple: ship a trusted semantic layer first, then grow it in lockstep with x402 and ERC-8004 adoption.

  • Done — already published and pinned.
  • In flight — being implemented or tested.
  • Planned — staged, with clear intent.

Legend

CommandLayer ships in phases. Each card calls out what’s done, in flight, or planned so it’s obvious where the protocol is today and what’s coming next.

Done In flight Planned

Phase 1 — Foundations of the semantic layer

Lock down the core language: a small Commons set, strict schemas, pinned artifacts, and a clear story that sits on top of x402 and ERC-8004.

Foundations

Commons v1.0.0

Done

10 universal Commons verbs with request/receipt schemas, shared primitives, and strict validation.

  • Analyze, classify, summarize, parse, format, clean, convert, describe, explain, fetch.
  • JSON Schema 2020-12, strict mode, no ad-hoc drift.
  • Pinned on IPFS with published checksums.
Infrastructure

Immutable protocol artifacts

Done

Make the language durable: immutable IDs, pinned CIDs, and repeatable builds.

  • Canonical $id layout for all schemas.
  • Manifest + checksums for clients and registries.
  • CI validation before anything is tagged.
Positioning

Semantic layer framing

In flight

Make it explicit: x402 moves money, ERC-8004 finds agents, CommandLayer defines what they do.

  • Docs and site updated to “semantic layer on x402 + ERC-8004.”
  • Clear intro for standards bodies and ecosystem partners.

Phase 2 — Commercial verbs & Agent Cards

Extend beyond analysis verbs into economic flows and discoverability, without losing the stability of the Commons layer.

Economic flows

Commercial verbs v1

In flight

Define the first Commercial verbs focused on money movement and fulfillment.

  • Initial set: checkout, pay, refund, ship, and related verbs.
  • Separate namespace and licensing from Commons.
  • Schema patterns aligned with x402 payment envelopes.
Identity & routing

Agent Cards v1

In flight

JSON descriptors that bind ENS names to verbs, schema versions, and x402 entrypoints.

  • Minimal spec for “what this agent does,” not how it’s implemented.
  • Compatible with ERC-8004 agent discovery.
  • Used by registries and clients to route calls.
Examples

Reference agents & flows

Planned

Concrete examples that show how to wire the semantic layer into real agents.

  • Reference summarize agent with full request/receipt flow.
  • Sample x402 endpoints using Commercial verbs.
  • End-to-end walkthroughs for implementers.

Phase 3 — Scaling out hundreds of verbs

Once the foundations are stable, widen the language: more Commons verbs, more Commercial verbs, and a clear cadence so builders can plan around them.

Coverage

Expanded Commons set

Planned

Add new Commons verbs where the ecosystem clearly converges on shared semantics.

  • Careful criteria for what qualifies as a Commons verb.
  • Focus on analysis, transformation, retrieval, and orchestration.
  • Versioned releases to avoid fragmenting the language.
Economics

Commercial verb families

Planned

Group Commercial verbs into clear domains that map directly to real economic activity.

  • Payments, checkout, refunds, settlement.
  • Fulfillment, shipping, returns, logistics.
  • Identity, compliance, risk, access control.
Scale

Hundreds of verbs staged

Planned

Roll out a larger library of verbs over time, instead of dumping everything at once.

  • Publish a public verb backlog for builders.
  • Prioritize high-utility verbs as x402 and ERC-8004 usage grows.
  • Keep the semantic layer coherent and opinionated.

Phase 4 — Ecosystem integrations & governance

Move from “one maintainer” to “trusted standard” without losing the discipline that makes the language useful.

Ecosystem

Integrations & tooling

Planned

Make it easy for anyone building on x402 and ERC-8004 to adopt CommandLayer by default.

  • Language bindings and SDK helpers for Commons verbs.
  • Tooling to generate clients from schemas and Agent Cards.
  • Examples that plug into existing agent frameworks.
Stewardship

Governance evolution

Planned

Start centralized for speed and correctness, then open up carefully.

  • Clear process to propose, debate, and adopt new verbs.
  • Security review and change control for the Commons layer.
  • Room for ecosystem-specific extensions without fragmenting the core.
Principle: CommandLayer is not trying to own every agent — it’s defining the shared language that makes agents composable. Governance is about keeping that language coherent as more builders and institutions adopt it.

Where builders fit in

You don’t need to wait for “all phases” to land. One verb, one Agent Card, or one example agent is enough to plug into the semantic layer today.

Start small:

  • Pick a Commons verb and wire a single agent to it.
  • Expose it via x402 when you’re ready to handle payment.
  • Describe it with an Agent Card so ERC-8004 style registries can find it.

As the roadmap advances — more Commercial verbs, more Cards, more tooling — the same semantic foundation compounds. The goal is straightforward: when people talk about agents doing work over x402 and ERC-8004, they use CommandLayer verbs by default.
