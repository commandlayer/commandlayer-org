Getting started with CommandLayer

Three moving parts: Commons verbs, schemas, and Agent Cards. This page shows how to call a verb, validate payloads, and describe your own agent in a way others can discover — on top of x402 and ERC-8004.

Protocol v1.0.0
Commons + Commercial + Agent Cards
NPM: @commandlayer/protocol-commons

How to think about the protocol

At minimum, you only need two things to integrate: a verb and its schemas. If you also want others to discover your agent, you publish an Agent Card.

1. Pick a verb

Choose a verb from Commons (e.g. summarize, analyze). Commercial v1.0.0 currently covers a small set of economic verbs (e.g. checkout, purchase) and you only need it if your flow moves money or triggers fulfillment.

You are not inventing a new format. You’re opting into a shared one.

2. Use the schemas

Take the *.request and *.receipt JSON Schemas for that verb and use them as the contract between your client and your agent.

You can validate with any JSON Schema 2020-12 validator.

3. Publish an Agent Card

If you want others to call you, publish an Agent Card JSON that says: “this ENS name serves these verbs at these x402 endpoints”.

Example: calling summarize

A typical integration is: client builds a summarize.request payload, validates it against the Commons schema, sends it to an agent endpoint, and receives a summarize.receipt.

1. Inspect the schemas

Both schemas are public:

summarize.request.schema.json
summarize.receipt.schema.json

JSON Schema 2020-12 version: v1.0.0

2. Shape a request

Minimal pseudo-example. The trace block is optional in Commons v1.0.0 and only needed if your agent uses tracing. 

{
  "x402": {
    "verb": "summarize",
    "version": "1.0.0",
    "network": "eip155:1"
  },
  "input": {
    "content": "Long article text here...",
    "content_type": "text/plain"
  },
  "constraints": {
    "max_tokens": 300,
    "style": "neutral"
  },
  "trace": {
    "trace_id": "c81f9f56-2c3a-4a6c-bc5b-8d5f8b372901",
    "span_id": "s1",
    "timestamp": "2025-11-26T00:00:00Z"
  }
}

3. Validate before sending

In Node.js with Ajv (example only): 

import Ajv from "ajv";

const ajv = new Ajv({ strict: true });

const schema = await fetch(
  "https://commandlayer.org/schemas/v1.0.0/commons/summarize/requests/summarize.request.schema.json"
).then(r => r.json());

const validate = ajv.compile(schema);

if (!validate(payload)) {
  console.error("Invalid summarize.request", validate.errors);
}

4. Send & receive a receipt

Where you send it depends on the agent. When reference agents are available, you’ll typically hit an x402-compatible HTTP endpoint listed in an Agent Card. 

const res = await fetch("https://<agent-endpoint>/summarize", {
  method: "POST",
  headers: {
    "content-type": "application/json"
  },
  body: JSON.stringify(payload)
});

const receipt = await res.json();
// validate against summarize.receipt.schema.json

Defining your own Agent Card

If you operate an agent and want others to discover and call it, you describe it with an Agent Card JSON file and host it at a stable URL under your control.

1. Start from the template

For Commons agents, cards live under: /agent-cards/v1.0.0/commons/<ens>.json

Example (structure only, not full spec): 

{
  "ens": "summarizeagent.eth",
  "owner": "commandlayer.eth",
  "contact": "dev@commandlayer.org",
  "verbs": [
    {
      "name": "summarize",
      "schema_version": "1.0.0",
      "tier": "commons"
    }
  ],
  "x402": [
    {
      "network": "eip155:1",
      "entry": "x402://summarizeagent.eth/summarize/v1"
    }
  ]
}

2. Host and reference it

Once the JSON file is in place (and ideally pinned on IPFS), you can:

  • Link to it from Agent Cards.
  • Reference it from ENS TXT records.
  • Share it with integrators and registries.

The important bit is that the card unambiguously ties an ENS name to verbs, schemas, and x402 entrypoints.

Commercial agents

Commercial agents follow the same pattern, typically under /agent-cards/v1.0.0/commercial/<ens>.json, with verbs such as authorize, checkout, purchase, ship, and verify, and corresponding x402 entries for paid flows.
