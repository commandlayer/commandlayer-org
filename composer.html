What this demo proves

Commons isn’t just a README — it’s a usable contract you can route through and validate.

  • You pick the verbs and order; the base receipt shape stays stable.
  • Each step returns a typed result + usage folded into a common receipt.base envelope.
  • A single trace_id threads the full flow.
  • Every receipt is validated with Ajv against receipt.base v1.0.0 (plus a minimal x402 envelope), mirroring the protocol-commons _shared schema.

Pick verbs, send a flow, inspect the receipts

Under the hood this calls /api/commons-flow on Vercel and returns receipts aligned with the shared receipt.base v1.0.0 schema used in Commons. For this demo, up to three steps share a single trace_id and each step’s receipt is validated with Ajv before it’s returned.
