What the Commons layer is

Commons defines a small, stable set of verbs that most agents will need: analyze, classify, fetch, summarize, and more. Each verb is pinned to a pair of schemas so clients and agents speak the same language.

Canonical verbs

10 verbs that cover core agent behaviors like analysis, summarization, formatting, and retrieval, without picking winners on implementation.

Request/receipt pairs

For every verb there is a *.request and *.receipt schema so calls are traceable, idempotent, and easy to validate.

Aligned with x402 / 8004

Schemas are designed to fit inside x402 envelopes and work cleanly with ERC-8004 style agent discovery and routing.

Stability & governance

Commons v1.0.0 is intentionally small and stable. Changes, if any, happen in new versions (v1.1.0, v2.0.0) so agents can rely on these schemas long-term.

During the v1.0.0 cycle, the Commons set is curated under a single-maintainer model to keep the command language coherent and predictable for early builders. Broader participation and formal governance can layer on without breaking this version.
