Agent Capabilities

Verifiable capabilities for machine-executed work

CommandLayer capabilities define what agents can do, how those actions are discovered, and how their receipts are verified.

Each capability can include schemas, examples, agent cards, ENS records, ERC-8004 registration, MCP/OpenAPI discovery, SDK support, and public verification URLs.

How capabilities work

  1. Action semantics define what the capability means.
  2. Request schemas define valid input.
  3. Execution produces output.
  4. Receipt schemas define the proof object.
  5. Discovery records expose the capability to agents, tools, wallets, and registries.
  6. VerifyAgent checks schema, hash, signature, and signer identity.
