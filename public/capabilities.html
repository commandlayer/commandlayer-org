Agent Capabilities

Verifiable capabilities for machine-executed work

CommandLayer capabilities define what agents can do, how those actions are discovered, and how their receipts are verified.

Try the verifier Wrap your agent

Each capability can include schemas, examples, agent cards, ENS records, ERC-8004 registration, MCP/OpenAPI discovery, SDK support, and public verification URLs.

1

Semantics

Action semantics define what the capability means.

2

Request schema

Request schemas define valid input before execution.

3

Execution

Execution produces output from an agent, runtime, API, or workflow.

4

Receipt schema

Receipt schemas define the proof object emitted after execution.

5

Discovery

Discovery records expose capabilities to agents, tools, wallets, and registries.

6

Verification

VerifyAgent checks schema, hash, signature, and signer identity.

Capability Map

One map for actions, discovery, and proof.

Trust & Verification is the active v1 capability group. The other groups represent namespace packages that can grow into schemas, agent cards, discovery records, and verifier support.
