Integrations

Connect external rails to verifiable CommandLayer receipts.

How integrations fit

CommandLayer does not replace wallets, payment protocols, webhook providers, or agent frameworks. It wraps external events and actions into signed, traceable CLAS receipts.

metadata.trace connects multi-step workflows. metadata.proof makes receipts verifiable.

Integration surfaces

Coinbase CDP Webhooks

Verified Coinbase CDP webhook events can be normalized into CLAS observe receipts.

Trust boundary: Coinbase HMAC authenticates the webhook to the receiving server; CommandLayer signing makes the normalized event publicly portable.

GitHub doc GitHub example

x402 Paid Actions

x402 handles payment negotiation/settlement; CommandLayer receipts prove the requested action, execution result, and signed output.

Trust boundary

  • Coinbase HMAC proves authenticity to the receiving server, not public verification by itself.
  • x402/payment rails prove payment settlement, not agent execution by themselves.
  • CommandLayer receipts begin public portability once the runtime signs the normalized event/action.
  • metadata.trace links the event, payment, action, and verification spans.
