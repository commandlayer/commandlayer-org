Roadmap

Execution is already live. The roadmap hardens it.

CommandLayer is past the “does this exist?” stage. Commons v1.1.0, Commercial v1.0.0, and Agent Cards v1.0.0 are released. Runtime is already live as a reference proving surface. The next phases deepen policy, metering, guarantees, and multi-runtime execution around the contract surface that already exists.

How to read this roadmap

This is not a future-existence roadmap. It is an operational roadmap layered on top of released contracts and a live reference runtime. Commons and Commercial define the contract. Agent Cards define discovery. Runtime defines execution.

Canonical receipt first

1. Released foundations

The protocol core is already published. Judges should read this as shipped architecture, not aspiration.

Commons v1.1.0

Released

Canonical public-goods request and receipt schemas for the base verb set.

  • Strict JSON Schema contracts for requests and canonical receipts.
  • Pinned artifacts and checksums for deterministic validation.
  • Public builder target for interoperable integrations.

Commercial v1.0.0

Released

Economic verbs for payments, checkout, fulfillment, shipping, and verification on the same schema discipline.

  • Commercial contracts stay public and portable.
  • Aligned with the broader x402 / agent commerce landscape without competing with transport rails.
  • Portable canonical receipts remain first-class.

Agent Cards v1.0.0

Released

Open identity and routing descriptors that let clients discover capabilities without guessing.

  • ENS-linked discovery and routing metadata.
  • Machine-readable capability declarations.
  • Identity layer pinned to canonical artifacts.

2. Live reference execution now

Runtime is not hypothetical. It is already live in reference form and proving the execution thesis today.

Reference proving surface

Live now

Execute published contracts, return canonical receipts, and optionally attach runtime proof and trace metadata.

  • Canonical receipt first, runtime metadata second.
  • Execution proves the contract can survive beyond a static schema repo.
  • Reference-grade surface for demos, validation, and protocol credibility.

Current operational focus

In flight

Move from reference execution toward hardened infrastructure without changing the contract layer underneath.

  • Policy enforcement and request classification.
  • Metering, throughput control, and idempotent execution flows.
  • Guarantee surfaces that justify commercial execution value.

3. In-flight operational hardening

These are the next real steps: more operational depth around a stack that already ships.

Policy and metering

In flight
  • Execution policy, retries, backpressure, and abuse controls.
  • Usage metering for commercial and managed runtime surfaces.
  • Clear separation between public contracts and paid operational guarantees.

Proof and trace discipline

In flight
  • More durable runtime metadata and provenance surfaces.
  • Structured trace attachments that do not mutate canonical receipt meaning.
  • Operational observability for managed execution providers.

Commercial hardening

Next
  • SLA-style guarantees and priority routing.
  • Managed commercial service surfaces around execution quality.
  • Support for more production-grade fulfillment and verification flows.

4. Planned ecosystem expansion

The next layer grows the ecosystem around the same released contract system rather than replacing it.

Multi-runtime execution

Planned
  • More runtime implementations honoring the same public schemas.
  • Competitive execution markets on top of shared meaning.
  • No lock-in at the contract layer.

Registry and routing expansion

Planned
  • Richer Agent Card indexing and discovery patterns.
  • More explicit routing across providers and commercial surfaces.
  • Cleaner integration paths for standards-adjacent ecosystems.

Broader contract coverage

Planned
  • Careful expansion of live schemas without turning reserved namespace into fake surface area.
  • Manifest remains strategic namespace, not the builder contract.
  • Build against published Commons and Commercial schemas, not reserved verbs.
