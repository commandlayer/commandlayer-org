Repositories

CommandLayer is a layered protocol stack: contract, discovery, execution, and tooling. Each repository owns a precise place in that architecture so canonical contracts stay distinct from routing metadata and execution metadata.

Contract and discovery

Read the stack in order: canonical contracts first, discovery metadata second.

schemas

protocol-commons

Canonical public contract base truth. Commons v1.1.0 defines the minimum verifiable receipt and the portable request/receipt schemas builders should learn first.

MIT JSON Schema IPFS-pinned
schemas

protocol-commercial

Economic contract layer extending Commons. Commercial v1.0.0 carries stable schemas for authorize, checkout, purchase, ship, and verify without monetizing semantics.

Apache-2.0 JSON Schema x402-aligned
schemas

agent-cards

Identity, discovery, and routing metadata for agents. Agent Cards describe identity, capabilities, schema versions, and entrypoints without redefining receipt semantics.

Apache-2.0 ENS IPFS-pinned
