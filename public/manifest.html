Quickstart: call an agent, get a receipt, verify it.

CommandLayer standardizes what agents do by pinning request/receipt schemas to canonical ENS names. You keep the same contract as runtimes, facilitators, and payment rails evolve.

  • Pick a verb (e.g. analyze), call the runtime endpoint.
  • Receive a structured receipt with cryptographic proof.
  • Validate against pinned schemas and verify signatures independently.

What you’ll accomplish

In under 5 minutes you’ll make a real request to a CommandLayer runtime, receive a signed receipt, and confirm the receipt is valid.

  1. Call POST /analyze/v1.0.0 with an x402 envelope.
  2. Inspect the receipt: status, result, trace, metadata.proof.
  3. Verify: schema ok, hash ok, signature ok.
  4. Resolve: use ENS TXT records to discover schema URLs & entry.

Step 1 — Call a live agent runtime

This is a real runtime endpoint. You send an x402 envelope (identity) plus input. The runtime returns a receipt you can validate and verify.

Copy/paste curl

Example: analyze (Commons v1.0.0) 

curl -sS --max-time 20 \
  -X POST 'https://runtime.commandlayer.org/analyze/v1.0.0' \
  -H 'Content-Type: application/json' \
  --data-binary '{
    "x402": {
      "entry": "x402://analyzeagent.eth/analyze/v1.0.0",
      "verb": "analyze",
      "version": "1.0.0"
    },
    "actor": "quickstart.commandlayer.org",
    "trace": { "trace_id": "quickstart-001" },
    "input": { "content": "hello world" }
  }'

Step 2 — Understand the receipt

A CommandLayer receipt is evidence, not a log. It includes the verb identity, trace, result payload, and a cryptographic proof block.

Identity

Receipts carry x402.entry, verb, and version so callers can route, cache, and replay safely across runtimes.

Trace

Traces link multi-step flows and allow attribution. Use one shared trace_id across steps to group receipts.

Proof

metadata.proof includes the hash and signature. Third parties can verify receipts offline.

Receipt excerpt

{
  "status": "success",
  "x402": { "entry": "x402://analyzeagent.eth/analyze/v1.0.0", "verb": "analyze", "version": "1.0.0" },
  "trace": { "trace_id": "trace_..." },
  "result": { "summary": "...", "insights": ["..."], "score": 0.1 },
  "metadata": {
    "proof": {
      "alg": "ed25519-sha256",
      "canonical": "json-stringify",
      "signer_id": "runtime.commandlayer.eth",
      "hash_sha256": "…",
      "signature_b64": "…"
    }
  }
}

Step 3 — Resolve schemas and entry via ENS

Each agent publishes machine-readable TXT records. That’s how discovery tools and clients find the canonical entrypoint and schema URLs.

Example ENS TXT records (typical)

cl.verb                  analyze
cl.version               1.0.0
cl.class                 commons
cl.entry                 x402://analyzeagent.eth/analyze/v1.0.0
cl.schema.request         https://commandlayer.org/schemas/v1.0.0/...
cl.schema.receipt         https://commandlayer.org/schemas/v1.0.0/...
cl.cid.schemas           bafy...
cl.schemas.mirror.ipfs   ipfs://bafy...
cl.schemas.checksums.ipfs ipfs://.../schemas.v1.0.0.checksums.txt
cl.agentcard             https://commandlayer.org/agent-cards/...
cl.cid.agentcards        bafy...
cl.owner                 commandlayer.eth
cl.receipt.alg           ed25519
cl.receipt.signer_id     runtime.commandlayer.eth

Step 4 — Verify a receipt (Node.js)

Verification should be a boring, deterministic check: schema → hash → signature. The signer public key is published via ENS / agent metadata.

Install dependencies

npm install tweetnacl tweetnacl-util

Verify code (minimal)

import nacl from "tweetnacl";
import { decodeBase64 } from "tweetnacl-util";
import crypto from "crypto";

// NOTE: receipt.metadata.proof.signature_b64 must NOT be included in the hashed payload.
// Canonicalization must match the runtime ("canonical" field in proof).

export function verifyReceipt(receipt, publicKeyBytes) {
  const proof = receipt?.metadata?.proof;
  if (!proof?.signature_b64 || !proof?.hash_sha256) return { ok: false, reason: "missing proof" };

  // clone + remove signature before hashing
  const unsigned = JSON.parse(JSON.stringify(receipt));
  delete unsigned.metadata.proof.signature_b64;

  // canonical bytes (current v1.0.0 uses JSON.stringify of the unsigned object)
  const canonical = JSON.stringify(unsigned);

  // hash sha256
  const hash = crypto.createHash("sha256").update(canonical, "utf8").digest("hex");

  if (hash !== proof.hash_sha256) return { ok: false, reason: "hash mismatch" };

  // verify ed25519 signature over the hash bytes
  const hashBytes = Buffer.from(proof.hash_sha256, "hex");
  const sigBytes = decodeBase64(proof.signature_b64);

  const ok = nacl.sign.detached.verify(hashBytes, sigBytes, publicKeyBytes);
  return ok ? { ok: true } : { ok: false, reason: "signature invalid" };
}
