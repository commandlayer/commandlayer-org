Systems can verify machine-action receipts automatically through URLs, badges, webhooks, SDKs, and agent-to-agent flows.

Ambient flow

Receipt reference → fetch receipt → verify checks → accept/reject action

The verifier can run inside a website, backend job, webhook consumer, SDK call, or agent-to-agent workflow.

Ambient verification does not require manual copy/paste.

Trust checks

  • schema_valid
  • hash_matched
  • signature_valid
  • signer_resolved
  • signer_matched
  • trust_verb
Schema-valid does not always mean cryptographically valid. A schema-valid receipt only means the receipt has the expected shape. Cryptographic validity still depends on hash, signature, and signer checks.

Developer use cases

webhook filtering

Accept only verified receipts before queuing follow-up work.

website badges

Display receipt status without opening manual verification tooling.

automated receipt checks

Run verification in scheduled jobs and continuous automation pipelines.

agent-to-agent validation

Require trust checks before one agent accepts another agent’s result.

audit trails

Store machine-action receipts as verifiable records for reviews.

dashboard/history pipelines

Feed verification outcomes into reporting and timeline systems.

CLAS is network-agnostic; ENS/ERC-8004 can assist discovery and identity but are not required for every verification flow.
