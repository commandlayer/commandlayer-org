Canonical Stack Proof Demo (production runtime verified)

This page documents the verified production runtime proof flow for CommandLayer. MCP local E2E now passes against the production runtime signer endpoint.

Canonical flow

MCP sign request → runtime signs receipt → verifier validates → tampered receipt invalidates

Honest caveat: MCP is a bridge, not the signer. The runtime signs. The verifier validates.

Production proof outcome

  • STEP 1: SIGNED
  • STEP 2: VERIFIED
  • STEP 3: TAMPERED INVALID

Verification response includes hash_matches=true and signature_valid=true. Tampered response includes hash_matches=false, signature_valid=false, and signature_error=hash_mismatch.
