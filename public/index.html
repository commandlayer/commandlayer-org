▸ not output — evidence

Execution ends with
proof, not a claim.

The artifact the system leaves behind: agent identity, routed endpoint, signer, proof hash, and verification status in one inspectable object.

1
Call the verb endpoint
POST to any Commons or Commercial verb endpoint. The runtime resolves the ENS-bound agent and executes the action.
2
Receive the canonical receipt
Every response carries a receipt: verb identity, status, and verb-specific output fields — flat, self-contained, schema-validated.
3
Verify independently
SHA-256 the canonical receipt bytes, check the Ed25519 signature from the proof block. Resolve the signer via ENS. Done.
4
Proof is portable
The receipt survives outside the runtime. Store it, share it, replay it — the hash still matches, the signature still verifies.
