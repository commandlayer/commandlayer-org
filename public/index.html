About

A protocol that makes agent meaning trustable, portable, and enforceable.

Most “agent systems” are demos glued together by private prompt formats and one-off APIs. CommandLayer is the opposite: canonical verbs, strict schemas, verifiable identity, and receipts — so agents can interoperate across runtimes, vendors, and payment rails without breaking.

What CommandLayer actually is

A semantic contract for autonomous agents — verbs + schemas + identity + receipts — designed so execution can be swapped without semantic chaos.

  • Commons: public-good verbs + schemas (free forever).
  • Commercial: same openness, but execution is metered.
  • Agent Cards: identity + discovery bindings for routing.
  • Runtime: where policy, guarantees, and economics live.

ENS Names — Interface Anchor

In CommandLayer, an ENS name is a single interface anchor that binds multiple protocol responsibilities into one verifiable reference.

Interface Anchor
ENS NAME
Everything else fans out from this.

Human-readable entry point

First place a human or agent looks. Stable, memorable invocation surface.

This replaces registries, directories, and service catalogs.

Schema authority

ENS TXT points to canonical schemas (CID / IPNS) that define request shape, receipt format, and constraints.

This is the semantic contract. Without it, nothing interoperates.

Validator reference

Schemas act as shared validation authority. Any agent can independently validate inputs, outputs, and receipts.

The ENS name becomes the verifier pointer: “did this conform to this command?”

Trust root

A long-lived name + stable pointer means trust accrues to the ENS name over time (versions, pins, receipts).

Like a public key accumulating reputation.

Version boundary

ENS TXT resolves to a versioned manifest (v1, v2…). Agents may pin to a version or float to latest compatible.

Silent breaking changes are prevented by design.

Receipt namespace

Every receipt binds ENS name + schema hash + version, enabling auditability, replay safety, and cross-runtime verification.

Receipts are evidence, not logs.

Governance handle

ENS ownership governs schema evolution and version publishing. Later upgradeable to multisig, timelock, or DAO.

Early authority without premature governance complexity.

Economic surface (future)

Even if dormant today, the surface exists: metered calls, certified execution, reputation, stake-to-publish, slashing for invalid receipts.

ENS becomes the billing + accountability unit.

One-sentence summary

An ENS name in CommandLayer is an interface anchor that bundles schemas, validation, versioning, trust, and future economics into a single verifiable reference point for autonomous agents.

CommandLayer Economic Rule (Final)

This is the whole economic system, with no ambiguity. If someone misunderstands CommandLayer, it’s because they don’t have this sentence in their head.

Economic Rule

Commons executes free.
Commercial is taxed.

FREE schemas, identity, discovery, Commons execution
TAXED commercial execution, throughput, guarantees
NEVER TAXED meaning, validation, interoperability

Protocol-Commons

Public goods layer. No rent. No tolls. No “open core” tricks.

  • Schemas: FREE
  • Agent-Cards: FREE
  • Discovery (ENS): FREE
  • Runtime execution: FREE (best-effort, reference-grade)

Commons exists to prove semantics, maximize adoption, eliminate friction, and create trust.

Protocol-Commercial

Same openness — the tax is only on execution when money moves.

  • Schemas: FREE
  • Agent-Cards: FREE
  • Discovery: FREE
  • Runtime execution: TAXED

The tax applies to execution of commercial verbs, throughput / volume, SLA guarantees, priority or certified execution, and leasing of verb-bound agents.

Where the tax is not

The tax is not hidden. It is not embedded in standards. It is not in the schema, the Agent-Card, ENS, or discovery.

Meaning stays neutral and portable.

Where the tax lives

The tax lives at execution time inside runtimes — because that’s where enforcement is natural and clean: metering, guarantees, priority, certified delivery, and throughput control.

Standards-clean. Regulator-clean. Developer-trustworthy.

Enforcement model

No semantic fork. No schema gating. No hidden rules.

Runtime enforces:

Lead sentence (use this everywhere): CommandLayer makes meaning free and taxes execution when money moves.

Why this architecture works

Most protocols try to tax access. CommandLayer taxes outcomes. That’s the difference between “standards drama” and “standards gravity.”

  • You avoid paywalled standards.
  • You avoid “open core” backlash.
  • You align incentives with real value.
  • You monetize where enforcement is natural.
  • Semantics remain neutral — interoperability doesn’t get held hostage.

This is now a coherent economic protocol, not just tech.
