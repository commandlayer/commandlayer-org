Getting started

Use CommandLayer without changing your stack.

CommandLayer does not run agents. It defines verbs, schemas, and discovery rules so you can build agents that interoperate while keeping your own infra, runtimes, and models.

1. Choose a verb

Start from a canonical verb your agent should support, for example summarize or verify.

You can discover available verbs via ENS TXT records or the static lookup JSON at /lookup/verbs.json.

2. Resolve canonical schemas

Each verb has a request and receipt schema under /schemas/v1.0.0/<layer>/<verb>/. 

schemas/
v1.0.0/
  commons/
    summarize/
      requests/summarize.request.schema.json
      receipts/summarize.receipt.schema.json

Your agent accepts payloads that validate against the request schema and returns receipts that validate against the receipt schema.

3. Bind your own endpoint

Implement your own endpoint (for example https://api.yourdomain.com/agents/summarize) that speaks the CommandLayer schema. You keep:

  • your own infra and runtimes,
  • your own auth and billing,
  • your own rate limits and SLAs.

CommandLayer only standardizes the message formats and discovery, not the execution.

4. Optionally publish an agent-card

Agent-cards describe identity and capabilities. A typical card:

  • binds an ENS name to one or more verbs,
  • declares the schemas it implements,
  • lists x402 and HTTPS entrypoints.

Publishing a card is optional but recommended if you want others to route to your agent in a standardized way.
