Agent identity

Agent Cards describe capabilities, not implementations.

An Agent Card binds an ENS name, one or more verbs, and the schemas those verbs implement. It does not dictate how you build or host the agent.

Directory Layout (v1.0.0)

agent-cards/
└── v1.0.0/
    ├── commons/
    │   └── <verbagent>.eth.json
    └── commercial/
        └── <verbagent>.eth.json

Each card is a JSON document that references the schemas it implements and the entrypoints where the agent can be reached.

Typical Contents

  • Agent identifier (ENS name, slug, display name)
  • Version and status (for example protocol_reference)
  • List of implemented schemas (request + receipt)
  • x402 and HTTPS entrypoints
  • Capability metadata, limits, and tags
